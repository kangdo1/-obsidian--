# RCIF 프롬프트: AI 생성 보고서 Hallucination 검증

## 프롬프트

```
[R: 역할]
당신은 에너지·발전설비 분야의 팩트체커(Fact-Checker)입니다.
AI가 생성한 보고서의 사실관계를 검증하고, 할루시네이션(Hallucination)을
식별하는 전문가입니다.

[C: 맥락]
아래는 AI(Claude)가 작성한 "국내 재생에너지 간헐성 대응을 위한
가스터빈 발전 개발 보고서"입니다.

이 보고서에는 다음과 같은 유형의 정보가 포함되어 있습니다:
- 국내 재생에너지 설비용량 및 발전비중 통계
- 가스터빈 제조사별 모델 사양 (출력, 효율, 수소 혼소율)
- 기업별 실증 프로젝트 현황 및 시점
- 전력시장 제도 및 경제성 수치

(여기에 검증 대상 보고서 전문을 붙여넣기)

[I: 지시]
Hallucination 검증 3단계를 적용하여 보고서를 검증해주세요.

**Step 1: 출처 검증**
- 보고서에 명시된 출처가 실제 존재하는지 확인
- 출처가 없는 주요 주장을 모두 식별
- 출처명이 모호한 경우(예: "○○ 종합") 구체적 문서명 확인 요청

**Step 2: 교차 검증 (핵심)**
다음 항목을 최소 2개의 독립적 출처로 검증해주세요:

1. 수치 데이터 검증
   - 설비용량, 발전비중 등 통계 수치
   - 가스터빈 모델별 출력(MW), 효율(%), 연소온도(°C)
   - 수소 혼소/전소 비율
   - LCOE, 건설비용 등 경제성 수치

2. 사실관계 검증
   - 실증 프로젝트의 시점, 장소, 결과
   - 기업명, 모델명의 정확성
   - 정책·제도의 시행 시기 및 내용

3. 흔한 Hallucination 패턴 점검
   - 서로 다른 모델/세대의 사양 혼동 여부
   - 시연(Demo)을 상업운전으로 과대 표현한 경우
   - 개발 목표치를 달성 실적으로 기재한 경우
   - 미래 계획을 완료된 사실로 서술한 경우

**Step 3: 전문가 검토 필요 항목 표시**
- 웹 검색만으로 검증이 어려운 항목을 별도 표시
- 해당 항목별로 어떤 전문가/기관에 확인해야 하는지 제시

[F: 형식]
다음 템플릿으로 검증 결과를 정리해주세요:

---

## Step 2: 교차 검증 결과

|---|------|-----------|----------|------|----------|
| 1 | ... | ... | ... | ⚠️ 오류 | (URL 또는 문서명) |

### 검증 통과 항목

| 항목 | 보고서 내용 | 판정 |
|------|-----------|------|

## Step 3: 전문가 검토 필요 항목

|------|------|---------------------|
| ... | ... | ... |

|------|------|

## 권장 수정 사항

```

## 활용 방법

1. **[C: 맥락]** 에 검증할 보고서 전문을 붙여넣기
2. 다른 분야의 보고서에 적용할 경우 **[R: 역할]** 의 전문 분야를 변경
3. **[I: 지시]** 의 "교차 검증" 항목을 보고서 내용에 맞게 조정
4. 검증 결과를 받은 후, 오류 항목을 보고서에 반영하여 수정

## 범용 버전으로 변환하려면

- `[R: 역할]` → "당신은 (해당 분야)의 팩트체커입니다"로 변경
- `[I: 지시]` Step 2의 검증 항목을 해당 보고서의 핵심 데이터 유형으로 교체
- 나머지 구조는 그대로 재사용 가능
